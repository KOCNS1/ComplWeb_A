Exercice 06 - Encore des requêtes dynamiques

Dans la même continuté des exercices précédents, réalisons un formulaire dynamique qui ne demande pas de recharger la page pour avoir un résultat.

> Il existe un webcomic très connu sous le nom d'XKCD que vous pouvez regarder sur l'URL suivante: https://xkcd.com/. Le but de cet exercice est de prendre le formulaire ci-dessous pour charger dynamiquement une BD provenant de cette série basé sur la saisie utilisateur. Un utilisateur va ainsi saisir un numéro de BD dans l'input, et lorsqu'il appuyera sur le bouton, une requête sera effectuée pour récupérer les informations basé sur le numéro de BD saisi. Une fois les informations récupérées, exploitez son contenu pour afficher:
  • Son titre
  • Sa description (stockée dans l'attrubut "alt" de l'objet de retour)
  • Une image contenant la BD en question
Vous ferez usage de l'URL https://xkcd.vercel.app/?comic={comicId} pour récupérer les informations demandées. L'attribut comicId dans l'URL est simplement le numéro de BD que l'on souhaite récupérer. A l'heure de la rédaction de cet exercice, c'est un chiffre qui peut aller de 1 à 2741. Pensez à gérer le cas d'erreur lorsque l'utilisateur saisit un numéro de BD qui n'existe pas. Regardez le contenu de l'objet "Response" après un fetch pour tacler cette problématique
